Overview

Kestrel generates Phase-Locked Loop designs from a high-level specification. Given a target frequency range, jitter budget, loop bandwidth, and process node, it emits parametrically consistent Verilog-AMS models, SPICE netlists, KiCad schematics, and testbenches.

The PLL topology is the Maneatis self-biased charge-pump PLL (US5727038, expired 2015), featuring a 4-stage differential ring oscillator with replica-biased symmetric load delay cells.

Quick Start

Command line

# Generate a 1 GHz PLL on SkyWater 130nm
kestrel gen \
  --freq-min 500M --freq-max 1.5G \
  --jitter 2ps --loop-bw 5M \
  --ref-freq 50M --process sky130 \
  --output ./my_pll/

# Interactive GUI
kestrel gui --process sky130

Run the behavioral PLL simulation

cd sim

# Compile Verilog-A models to OSDI (one-time)
openvaf kes_vco.va --output kes_vco.osdi
openvaf kes_cp.va  --output kes_cp.osdi
openvaf kes_lf.va  --output kes_lf.osdi

# Run behavioral PLL
ngspice kes_run_beh.sp

Run the transistor-level VCO

cd sim
Xyce kes_vco_xyce.cir    # Sweeps Vctrl 0.3–1.5 V, measures frequency

Run VCO optimization

cd sim
python3 opt/optimize_vco.py --max-iter 100 --refit

Maneatis Delay Cell

The core VCO building block. Each output has a diode-connected PFET (sets common mode) and a Vctrl-gated PFET (variable conductance for frequency control). The self-biased replica adjusts the tail current to track Vctrl.

Design Engine

The engine (kestrel/design/engine.py) computes all PLL parameters analytically from the user specification:

  • Divider ratio from frequency range and reference clock
  • VCO Kvco from frequency range and control voltage swing
  • Charge pump current (50–200 µA, scaled by frequency)
  • Loop filter R, C1, C2 via Gardner's method for target bandwidth and phase margin
  • All transistor W/L sizing using square-law MOSFET equations
  • Jitter estimation from thermal noise model

Simulation

Behavioral (ngspice)

Uses OSDI-compiled Verilog-A models for VCO, charge pump, and loop filter. PFD and divider use XSPICE digital primitives (d_dff, d_and, adc/dac_bridge).

The VCO is a Van der Pol oscillator with f = kvco * V(vctrl). Self-starting via asymmetric bias term. Compiled from kes_vco.va with OpenVAF.

Files:
sim/kes_run_beh.sp (launcher)
sim/kes_vco.va, kes_cp.va, kes_lf.va

Transistor-level (Xyce)

4-stage differential Maneatis ring on sky130 BSIM4 binned models. Uses .STEP parameter sweep over Vctrl with .MEASURE for frequency extraction via differential zero-crossing.

Sky130 models: stripped subcircuit wrappers, direct bin references (bin 6 for L=0.36u W≥7u devices).

Files:
sim/kes_vco_xyce.cir
sim/models/sky130_*_xyce.spice

Refit

After optimization, --refit fits a polynomial f(Vctrl) = a3V3 + a2V2 + a1V + a0 to the transistor-level data and generates an updated Verilog-A behavioral model (opt/kes_vco_refit.va) that matches the layout exactly.

Key Observations

  • Behavioral model: linear Kvco ≈ 707 MHz/V across 0.1–1.5 V
  • Transistor level: Kvco ≈ 298 MHz/V, valid only 0.7–1.1 V
  • Frequency saturates above Vctrl ≈ 1.0 V (Vctrl PFET fully off)
  • No oscillation below 0.7 V (PFET load overwhelms diff pair)
  • Refit polynomial matches transistor curve within ±3.3%
  • Optimization: 112 evaluations, cost 0.363 → 0.249
